# Risk Register

## Risk 1 — scope drift
Mitigation: freeze non-goals early.

## Risk 2 — JSON output instability
Mitigation: define schemas deliberately and prefer additive changes.

## Risk 3 — artifact contract churn
Mitigation: preserve backward compatibility where possible.

## Risk 4 — duplicated surfaces
Mitigation: build the integration API around existing tested internals.

## Risk 5 — docs drift from code
Mitigation: tie docs/examples to actual tested paths.
