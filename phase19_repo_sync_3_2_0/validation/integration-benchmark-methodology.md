# Integration Benchmark Methodology

Validation v2 expands benchmarking beyond core runtime operations.

## Focus areas

- Python API export / inspect / verify / run
- CLI `--json` automation paths
- artifact contract v2 metadata inspection
- reproducible tamper-detection scenario

## Measurement model

Each benchmark is executed multiple times and recorded in milliseconds. The emphasis is reproducibility and automation suitability, not absolute performance claims across environments.

## Why this matters

Aether-9 v3.2.0 is positioned as an integration-ready platform. Validation therefore needs to show that external systems can call Aether programmatically and consume machine-readable responses consistently.
